## Summary

Migrate the full-stack Food Ordering System (both Express backend and React frontend) from JavaScript to TypeScript. The backend will configure `tsconfig.json` and run under `tsx` in local development with `tsc` compile steps for production build output. Mongoose models, Express request handlers, router middleware, React context hooks, and frontend views will be strongly typed using modern compiler checks.

## Technical Context

**Language/Version**: TypeScript 5+ (Targeting ESNext / Node.js 18+ / React 19)

**Primary Dependencies**: typescript, tsx, @types/node, @types/express, @types/cors, @types/jsonwebtoken, @types/bcryptjs, react-router-dom, axios

**Storage**: MongoDB (Mongoose schemas strongly typed with document interfaces)

**Testing**: Strict type-checking via `tsc --noEmit` and manual integration testing of checkout, webhooks, and admin operations

**Target Platform**: Node.js Linux server (Backend) / Web Browser (Frontend)

**Project Type**: Web application (separate client and server modules)

**Performance Goals**: Fast hot-reload recompilation (under 350ms) and type checking under 2 seconds

**Constraints**: Strict compiler settings (`strict: true`), correct ES Module format compatibility, correct type definitions for custom Express authentication middleware (`req.user`)

**Scale/Scope**: Refactoring all database models, controllers, routers, contexts, components, and views (~20 files total).

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- **Principle I: Stateless JWT Auth & Authorization** -> **PASS**. Middlewares (`auth` and `adminGuard`) will be typed to accept and extend standard Express Requests. Route protection components on the frontend (`PrivateRoute`, `AdminRoute`) will use TSX props validation.
- **Principle II: PayHere Sandbox Flow** -> **PASS**. Typing hash generation and webhook parameters to ensure strict MD5 signature formatting and prevent casting errors on webhook notifications.
- **Principle III: Strict Mongoose Schema and Database Consistency** -> **PASS**. Create document interfaces extending Mongoose schemas to guard mutations, schemas, and queries.
- **Principle IV: Clean Separation of Layered Architecture** -> **PASS**. TypeScript directories will mirror the established folder divisions cleanly: `models/`, `controllers/`, `routes/`, `components/`, and `pages/`.
- **Principle V: Data Seeding and Development Setup** -> **PASS**. The seed script will be run using `npx tsx seed.ts` to ensure consistency.
